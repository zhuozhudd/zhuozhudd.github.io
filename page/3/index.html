《剑指Offer》 56 - 数组中数字出现的次数

题目1：剑指 Offer 56 - I. 数组中数字出现的次数

一个整型数组 nums 里除两个数字之外，其他数字都出现了两次。请写程序找出这两个只出现一次的数字。要求时间复杂度是$O(n)$，空间复杂度是$O(1)$。

题解1：

知识点：

  • 异或运算：两个相同数字异或为0，且满足交换律，即将下方数列执行异或运算，留下的结果则为出现一次的数字x
    • a ^ b ^ b ^ c ^ x ^ a ^ c = a ^ a ^ b ^ b ^ c ^ c ^ x = x

解题思路：

  • 由异或运算的性质可知，对nums中所有数字执行异或运算，得到的结果是两个只存在一次的数字x1和x2的异或结果，即x1 ^ x2，令n = x1 ^ x2，若n的某二进制位是1，则x1和x2在此二进制位一定不同，基于此位将nums数组分成两部分，x1和x2分别在这两个子数组是唯一的，分别进行异或运算，即可将x1和x2分离。

题目2：剑指 Offer 56 - II. 数组中数字出现的次数 II

在一个数组 nums 中除一个数字只出现一次之外，其他数字都出现了三次。请找出那个只出现一次的数字。

题解2：

  • 建立一个长度为32的数组counts，遍历nums，将nums中各个数字的各个二进制位相加并统计在counts数组中，然后对counts数组中的各元素对 3取余，结果即为只出现一次的数字的二进制形式，将其恢复到数字res中即可。

《剑指Offer》 52 - 两个链表的第一个公共节点

题目：剑指 Offer 52. 两个链表的第一个公共节点

输入两个链表，找出它们的第一个公共节点。

如下面的两个链表：

在节点 c1 开始相交。

此题与LeetCode160相同：160. 相交链表

题解：

方法一：双指针法 $O(m+n) - O(1)$

  • 先逐个遍历两个链表，获得两个链表的长度，并求得两个链表长度的差值 dis
  • 令pA指针指向较长的那个链表，其长度为lenA
  • 从头重新遍历两个链表，先让 pA 在链表 A上走 dis 个节点。然后 pA 和 pB同时向前走，当两个指针指向同一个节点时，此节点即时两个链表的第一个公共节点

神之一手，令人惊叹的解法：

对于链表A，如果开始到公共节点的举例为 a，对于链表B，开始到公共节点的举例为 b，公共节点到结尾的距离均是 c

  • 指针pA从A的头结点出发，遍历到尾结点后指向链表B的头节点，继续向前走，走到公共节点，共走了：

$$(a+c)+b = a + b +c$$

  • 指针pB从B的头结点出发，遍历到尾结点后指向链表A的头节点，继续向前走，走到公共节点，共走了：

$$(b+c)+a = a + b +c$$

此时指针 pB和pA指向相同的节点，即第一个公共节点。

若两链表 无 公共尾部 ：指针 pA , pB 同时指向 nul

方法二：辅助栈 $O(m + n) - O(m + n)$（浪费空间 不推荐）

  • 把两个链表放到两个栈中，然后依次出栈比较两个栈顶元素，当两个指针不同，则上个循环中的栈顶元素为公共节点

《剑指Offer》 51 - 数组中的逆序对

题目：剑指 Offer 51. 数组中的逆序对

在数组中的两个数字，如果前面一个数字大于后面的数字，则这两个数字组成一个逆序对。输入一个数组，求出这个数组中的逆序对的总数。

题解：

归并排序与逆序对统计：

终止条件： 当 $left \ge right$ 时，代表子数组长度为 1，此时终止划分

递归划分： 计算数组中点 $mid$，递归划分左子数组 merge_sort (left, mid) 和右子数组 merge_sort(mid + 1, right)

合并与逆序对统计：

  • 暂存数组 $nums$ 闭区间 [i, right] 内的元素到辅助数组 temp
  • 循环合并：设置双指针 i 、j 分别指向左 、右子数组的首个元素
    • if i = mid + 1 时：代表左子数组已经合并完，因此添加右子数组当前元素 temp[j]，并 j++
    • else if j = right + 1时：代表右子数组已经合并完，因此添加左子数组当前元素temp[i]，并i++
    • else if temp[i] <= temp[j]时：添加左子数组当前元素 temp[i]，并i++
    • else temp[i] > temp[j]时：添加右子数组当前元素 temp[j]，并j++，此时构成 mid - i + 1个逆序对，统计添加至 res

返回值： res

简洁写法：

  • 以下两种情况可以合为一处：

    • else if j = right + 1时：代表右子数组已经合并完，因此添加左子数组当前元素temp[i]，并i++
    • else if temp[i] <= temp[j]时：添加左子数组当前元素 temp[i]，并i++

  • mergeSort 和 mergeAndCount 可以写成一个函数

  • 如果终止条件是 if (left >= right) return 0 ，写大于等于，可以省去 reversePairs()中的特判，即if (nums.size() < 2) return 0

题目：剑指 Offer 49. 丑数

把只包含质因子 2、3 和 5 的数称作丑数（Ugly Number）。求按从小到大的顺序的第 n 个丑数。

本题与LeetCode264相同：264. 丑数 II

题解：

方法一：三指针定界 + 动态规划 $O(N) - O(N)$

dp数组中的丑数是从小到大排序的，最大记作M，而接下来的丑数必定是之前的某一丑数乘以2 或 3 或 5得到的，对于乘以2而言，肯定存在某一个丑数 dp[a]，排在它之前的每个丑数乘以2的得到的结果都会小于M，在它之后的每个丑数乘以2得到的结果都大于M，这个下标 a 就是第一个界限，同理可得另外两个界 dp[b]和 dp[c]，所以最新求得的丑数，就是 dp[a] * 2, dp[b] * 3, dp[c] * 5 中的最小值，假设是dp[a] * 2，则 dp[i] = dp[a] * 2，界限 a 右移，a++

方法二：小根堆（优先队列）$O(N\log(N)) - O(N)$

最小堆初始为空，首先将最小的丑数 1放入最小堆中，每次取出堆顶元素 x，则 x是堆中最小的元素，然后将 2x , 3x , 5x 放入堆中。

用一个集合用来去重，往堆中加入丑数时判断之前是否已经加入过。

《剑指Offer》 48 - 最长不含重复字符的子字符串

题目：剑指 Offer 48. 最长不含重复字符的子字符串

请从字符串中找出一个最长的不包含重复字符的子字符串，计算该最长子字符串的长度。

此题与LeetCode3相同：3. 无重复字符的最长子串

题解：

一维动态规划问题：

状态定义：设动态规划列表 $dp$，$dp[i]$ 代表以字符 $s[i]$ 为结尾的最长不含重复字符串的子字符串长度

状态转移方程： 固定 $i$，从 $i$ 往 前找与 $s[i]$ 相同的字符 $s[j]$，距离为 $i - j$

  • 没有找到 s[j] , 且已经找到了字符串开头，则 $dp[i] = dp[i-1] + 1$
  • $dp[i-1] < i - j$ ，说明与$s[i]$ 相同的字符 $s[j]$ 在上个 $dp[i-1]$的区间之外，则 $dp[i] = dp[i-1] + 1$
  • $dp[i-1] \ge i - j$ ，说明与$s[i]$ 相同的字符 $s[j]$ 在上个 $dp[i-1]$的区间之内，则 $dp[i] = i - j$

返回值：$max(dp)$

注意事项：做动态规划题目前，写完状态转移方程后，想一想能否用常数个变量完成动态规划过程，从而避免开辟整个 $dp$ 数组，节省空间开销。

方法一：动态规划 + 向前遍历寻找 $s[j]$ ， $O(N^2) - O(1)$

方法二：动态规划 + 哈希表 $O(N) - O(1)$

字符的 ASCII 码范围为 0 ~ 127，哈希表 dic 最多使用 $O(128) = O(1)$大小的额外空间，所以空间复杂度为 $O(1)$，哈希表存放着最近一次出现字符的所在位置。

举例说明代码中第一次出现某字符 j = -1 的原因，例如字符串”aba”：

  • 当i = 0 时，mp还是空的，当然找不到 "a"，所以j = -1，后面 i - j = 0 - (- 1) = 1，dp_temp在 i = 0 处等于 1，res = 1

  • 当 i = 1时，mp中只有"a"，找不到 "b"，所以 j仍然是-1，后面i - j = 1 - (-1) = 2，dp_temp在 i = 1 处等于2，res = 2

  • 当 i = 2 时，mp中已经有了"a"，mp["a"] = 1，j = 1，然后更新一下，令当前的索引 i = 2 赋值给 mp["a"]，这样mp["a"]就记录当前 "a"的出现位置，j 记录了上一个 "a" 出现的位置，i - j = 2 - 1 = 1 < res，dp_temp在i = 2处等于1，res = 2

方法三：哈希表 + 滑动窗口 $O(N) - O(1)$

  • 哈希表记录字符s[i] 最后一次出现的索引
  • 每轮更新左边界 j，保证区间 [j + 1, i] 内无重复字符且最大
  • 每轮更新结果res，取上一轮的res与本轮双指针区间 [j + 1, i]的宽度 i - j比较，取最大值

《剑指Offer》 47 - 礼物的最大值

题目：剑指 Offer 47. 礼物的最大价值

在一个 m*n 的棋盘的每一格都放有一个礼物，每个礼物都有一定的价值（价值大于0）。你可以从棋盘的左上角开始拿格子里的礼物，并每次向右或者向下移动一格、直到到达棋盘的右下角。给定一个棋盘及其上面的礼物的价值，请计算你最多能拿到多少价值的礼物？

题解：

二维动态规划问题：

状态定义：设动态规划矩阵 $dp$，$dp(i, j)$ 代表从期盼的左上角开始，到达单元格 $(i ,j)$ 时能拿到礼物的最大累积价值

状态转移方程：

  • $i = 0 , j = 0$ 时，为起始元素
  • $i = 0, j \ne 0$ 时，为矩阵第一行元素，只可从左边径直向右边走
  • $i \ne 0, j = 0$ 时，为矩阵第一列元素，只可从上径直向下走
  • $i \ne 0, j \ne 0$ 时，可从左或上边到达到此处

初始值： $dp[0][0] = grid[0][0]$

返回值： $dp$ 矩阵右下角元素

方法一 ：初级写法，有诸多可优化之处

方法二：多开辟一行一列数组，可以使代码更加简洁

  • 方法一的循环中，之所以要分 i == 0 和 j == 0的情况讨论，是因为如果不讨论， dp[i][j] = max(dp[i-1][j], dp[i][j-1]) + grid[i][j] 中的 i - 1和j - 1会产生越界。如果多开辟一行一列数组从 dp[1][1] 开始，就不需要考虑越界问题，使代码更加简洁。
  • 要注意 dp 数组做了偏移，但是原 grid 数组并没有，所以在循环中是和 grid[i-1][j-1]相加。

方法三：$dp[i][j]$ 只与 $dp[i-1][j], dp[i][j-1], grid[i][j]$有关系，可使用 $grid$ 矩阵用作 $dp$ 矩阵，在原矩阵上修改，可使用空间复杂度从 $O(MN)$ 降低至 O(1)

方法三改进：当 $grid$ 矩阵很大时， $i = 0$ 或 $j = 0$ 的情况仅占极少数，相当循环每轮都冗余了一次判断。因此，可先初始化矩阵第一行和第一列，再开始遍历递推。
