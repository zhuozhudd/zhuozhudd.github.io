7. Reinforcement Learning Gym Envs

在“pip install pybullet”期间安装了一套RL Gym环境。这包括PyBullet版本的OpenAI Gym Env，如ant, hopper, humanoid和walker。还有一些环境适用于模拟和真实的机器人，如Ghost
Robotics Minitaur quadruped，麻省理工赛车和库卡机器人手臂抓取环境。


pybullet, pybullet_envs, pybullet_data的源代码和示例: https://github.com/bulletphysics/bullet3/tree/master/examples/pybullet/gym.



可以使用RL训练算法来训练环境，例如DQN、PPO、TRPO和
DDPG。以下是一些经过训练的例子:
python -m pybullet_envs.examples.enjoy_TF_HumanoidBulletEnv_v0_2017may
python -m pybullet_envs.examples.kukaGymEnvTest

9. Debug GUI, Lines, Text, Parameters

PyBullet有一些功能，可以更容易地调试、可视化和调优模拟。
这个功能只有在有一些3D可视化窗口时才有用，例如GUI模式或连接到单独的物理服务器(例如“物理服务器”模式的浏览器或独立的OpenGL GUI物理服务器)。

9.1 addUserDebugLine, Text, Parameter

  • addUserDebugline
    • 可以添加由3d起始点(从)和结束点(到)指定的3d线条，以及颜色
      (红、绿、蓝)，线宽和持续时间(以秒为单位)。
    • addUserDebugLine将返回一个非负的唯一id，它允许使用removeUserDebugItem删除这个line。(当使用’replaceItemUniqueId’参数时，它将返回replaceItemUniqueId)。
Input Parameters type descriptions
R lineFromXYZ vec3f 直线在笛卡尔世界坐标中的起始点
R lineToXYZ vec3f 直线在笛卡尔世界坐标中的终点
O lineColorRGB vec3f RGB颜色[红、绿、蓝]范围内各分量
(0…1)
O lineWidth float 线宽(受OpenGL实现限制)
O lifeTime float 使用0为永久线路，或正时间秒
(之后会自动删除一行)
O parentObjectUniqueId int 即将到来的PyBullet 1.0.8新增功能:在parent object/link局部坐标中绘制直线。
O replaceItemUniqueId int 替换现有的line(以提高性能和避免移除/添加的闪烁)参见f10_racecar.py示例。
O physicsClientId int



  • addUserDebugText
    • 可以使用颜色和大小在特定位置添加一些3d文本
    • addUserDebugText将返回一个非负的唯一id，它允许你使用removeUserDebugItem删除行
    • See also pybullet/examples/debugDrawItems.py
Input Parameters type descriptions
R text text 表示为字符串(字符数组)的文本
R textPosition vec3f 文本在笛卡尔世界坐标中的三维位置[x, y, z]
O textColorRGB vec3f RGB颜色[红、绿、蓝]范围内各分量(0 . . 1)
O textSize float Text大小
O lifeTime float 为永久文本使用0，或以秒为单位的正时间(之后文本将自动删除)
O textOrientation vec4f 默认情况下，调试文本将始终面向摄像头，自动旋转。通过指定文本方向(四元数)，定向将固定在世界空间或局部空间(当parent指定时)。注意，不同的实现/着色器用于面向摄像机的文本，具有不同的外观:面向摄像机的文本使用位图字体，指定方向的文本使用TrueType字体。
O parentObjectUniquedId int 即将到来的PyBullet 1.0.8新增功能:在parent object/link的局部坐标中绘制直线。
O parentLinkIndex int 即将到来的PyBullet 1.0.8新增功能:在parent object/link的局部坐标中绘制直线。
O replaceItemUniqueId int 替换现有的文本项(以避免移除/添加的闪烁)
O physicsClientId int



  • addUserDebugParameter
    • ：addUserDebugParameter允许你添加自定义滑块和按钮来调整参数。它将返回一个唯一的id。这使用readUserDebugParameter读取参数的值。
Input Parameters type descriptions
R paramName string 参数名
R rangeMin float 最小值。如果最小值>最大值将出现一个按钮而不是滑块
R rangeMax float 最大值
R startValue float 开始值
O physicsClientId int



  • readUserDebugParameter
    • 对于滑块，返回值是该参数的最新读取值。对于按钮，每按一次按钮，按钮的getUserDebugParameter的值增加1。
Input Parameters type descriptions
R itemUniqueId int 由’addUserDebugParameter’返回的唯一id
O physicsClientId int


  • removeAllUserParameters

    • 这将删除所有滑块和按钮
    • | O | physicsClient | int | … |

  • removeUserDebugItem/All

    • 函数添加user debug lines ，如果成功，文本将返回一个非负的唯一id。可以使用removeUserDebugItem这个惟一id删除调试项
Input Parameters type descriptions
R itemUniquedId int 要删除的调试项的唯一id(line, text etc)
O physicsClientId int

  • removeAllUserDebugItems

    • 删除所有debug items (text, lines etc)

  • setDebugObjectColor

    • 内置的OpenGL可视化工具有一个线框调试渲染特性:按“w”键切换。
      线框有一些默认颜色。可以使用setDebugObjectColor覆盖特定对象的颜色和link
Input Parameters type descriptions
R objectUniqueId int Id
R linkIndex int link 索引
O objectDebugColorRGB vec3f 调试颜色为[红，绿，蓝]。如果不提供，自定义颜色将被删除。
O physicsClient int

9.2 addUserData

  • addUserData

    • 简而言之，添加、删除和查询用户数据，此时文本字符串，附加到主体的任何链接上。请参阅userData.py示例了解如何使用它。它将返回一个userDataId。注意，您还可以在urdf文件中添加用户数据。

  • getUserData

    • getUserData将根据addUserData返回的userDataId接收用户数据。
    • userData.py

  • syncUserData

    • syncUserData将在多个客户端更改用户数据(addUserData等)的情况下同步用户数据(getUserData)。

  • removeUserData

    • 给定userDataId, removeUserData将删除先前添加的用户数据。

  • getUserDataId and getNumUserData

    • getNumUserData将根据bodyUniqueId返回用户数据条目的数量。

  • getUserDataInfo

    • getUserDataInfo检索用户数据的键和标识符为(userDataId, key, bodyUniqueId, linkIndex, visualShapeIndex)

9.3 configureDebugVisualizer

  • configureDebugVisualizer
    • 可以配置一些内置OpenGL可视化工具的设置，例如启用或禁用线框图、阴影和GUI渲染。这是非常有用的，因为一些笔记本电脑或桌面gui与我们的OpenGL 3可视化工具有性能问题
    • pybullet.configureDebugVisualizer(pybullet.COV_ENABLE_WIREFRAME,1)
Input Parameters type descriptions
R flag int 要启用或禁用的功能:such as
COV_ENABLE_WIREFRAME,
COV_ENABLE_SHADOWS,
COV_ENABLE_GUI,COV_ENABLE_VR_PICKING,
COV_ENABLE_VR_TELEPORTING,
COV_ENABLE_RENDERING,
COV_ENABLE_TINY_RENDERER,
COV_ENABLE_VR_RENDER_CONTROLLERS,
COV_ENABLE_KEYBOARD_SHORTCUTS,
COV_ENABLE_MOUSE_PICKING,
COV_ENABLE_Y_AXIS_UP (Z is default world up axis),
COV_ENABLE_RGB_BUFFER_PREVIEW,
COV_ENABLE_DEPTH_BUFFER_PREVIEW,
COV_ENABLE_SEGMENTATION_MARK_PREVIEW
R enable int 0 or 1
O lightPosition vec3 visualizer的灯光位置
O shadowMapResolution int 阴影贴图纹理的大小，对于许多gpu来说通常是power of 2。默认是4096。现代gpu可以处理16384或32768或更高
O shadowMapWorldSize int 阴影地图在世界空间的大小(单位为米，默认为10)
O physicsClientId int

9.4 get/resetDebugVisualizerCamera

  • get/resetDebugVisualizerCamera

    • Warning:getDebugVisualizerCamera的返回参数与resetDebugVisualizerCamera的返回参数顺序不同。将在未来的API修订版(主要的新版本)中修复

  • resetDebugVisualizerCamera

    • 可以重置3D OpenGL调试visualizer摄像机距离(眼睛和摄像机目标位置之间)，摄像机偏航和俯仰和摄像机目标位置。
    • pybullet.resetDebugVisualizerCamera( cameraDistance=3, cameraYaw=30,
      cameraPitch=52, cameraTargetPosition=[0,0,0])
Input Parameters type descriptions
R cameraDistance float 从eye到camera目标位置的距离
R cameraYaw float 相机yaw(偏移)角度(以角度表示)左右
R cameraPitch float 摄像机Pitch(俯仰)角度(以度表示)向上/向下
R cameraTargetPosition vec3f cameraTargetPosition是相机的焦点
O physicsClientId int



  • getDebugVisualizerCamera
    • 可以得到相机的宽度和高度(以像素为单位)，它的视图和投影矩阵使用这个命令。输入参数是可选的physicsClientId,输出信息见下表
Output Parameters type descriptions
width int 相机图像的像素宽度
height int 相机图像的像素高度
viewMatrix float16,list of 16 floats 相机的view matrix
projectionMatrix float16,list of 16 floats 相机投影矩阵
cameraUp float3,list of 3 floats 相机的上轴，在笛卡尔世界空间坐标中
cameraForward float3,list of 3 floats 相机的前轴，在笛卡尔世界空间坐标
horizontal float3,list of 3 floats TBD。这是一个水平矢量，可以用来生成射线(例如，用于鼠标选择或创建一个简单的射线跟踪器)
vertical float3,list of 3 floats ：TBD。这是一个垂直矢量，可以用来生成射线(例如，鼠标选择或创建一个简单的射线追踪器)。
yaw float 相机的偏移角，在笛卡尔局部空间坐标中
pitch float 相机俯仰角，在笛卡尔局部空间坐标中
dist float 相机与相机目标之间的距离
target float3,list of 3 floats 相机的目标，在笛卡尔世界空间坐标



9.5 getKeyboardEvents, getMouseEvents

  • getKeyboardEvents
    • 调用getKeyboardEvents可以接收自上次调用以来发生的所有键盘事件,每个事件有一个键码和状态，状态是位标志KEY_IS_DOWN, KEY_WAS_TRIGGERED 和 KEY_WAS_RELEASED的组合，如果键盘是from up to down状态，会接收到KEY_IS_DOWN和KEY_WAS_TRIGGERED状态，如果键盘已经被按下，然后松开，会接收到KEY_IS_DOWN和KEY_WAS_RELEASED状态
    • 定义了一些特殊的键：B3G_F1 … B3G_F12, B3G_LEFT_ARROW,B3G_RIGHT_ARROW, B3G_UP_ARROW, B3G_DOWN_ARROW, B3G_PAGE_UP,B3G_PAGE_DOWN, B3G_PAGE_END, B3G_HOME, B3G_DELETE, B3G_INSERT,B3G_ALT, B3G_SHIFT, B3G_CONTROL, B3G_RETURN.
Input Parameters type descriptions
O physicsClientId int …




输出是一个keycode ‘key’和键盘状态’value’的字典。

  • getMouseEvents
    • 与getKeyboardEvents类似，可以获得自上次调用getMouseEvents以来发生的鼠标事件。所有鼠标移动事件合并到一个最新位置的单个鼠标移动事件中。此外，给定按钮的所有鼠标按钮事件将被合并。如果一个按钮向下和向上，状态将是’KEY_WAS_TRIGGERED ‘。对于鼠标按钮状态，我们重用KEY_WAS_TRIGGERED /KEY_IS_DOWN /KEY_WAS_RELEASED。
    • 输入参数只有 | O | physicsClient | int | … |
    • See createVisualShape.py for an example of mouse events, to select/color objects.
Output Parameters type descriptions
eventType int MOUSE_MOVE_EVENT=1, MOUSE_BUTTON_EVENT=2
mousePosX float 鼠标指针的x坐标
mousePosY float 鼠标指针的y坐标
buttonIndex int 鼠标左键/中键/右键索引
buttonState int ：flag KEY_WAS_TRIGGERED / KEY_IS_DOWN / KEY_WAS_RELEASED
