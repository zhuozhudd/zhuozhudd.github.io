二、版本追踪

2.1 log 追踪

  • git log
    • 修改记录日志
  • git log -p -2
    • 最近两次的修改记录,包括修改的内容
  • git log –author=’name’
    • 输出所有此作者所做的提交
  • git log –oneline
    • 只显示关键信息 哈希值 | 描述
  • git log –graph
    • 版本线图
    • 可以和online组合使用 git log –graph –oneline
  • git log –pretty=online
    • 完整哈希值 | 描述
  • git log –pretty=format 格式化输出
    • git log –pretty=format:”%h - %an, %ar : %s”
    • 输出 6ad963a - WallE, 3 hours ago : change index.html
    • 对应 –> 哈希值 - 作者 , 时间 : 描述

2.2 追踪文件修改前后的区别

  • git diff [file_name]
    • 显示更新的具体内容, 红色为旧内容,绿色为修改后的内容
  • git diff –staged
    • 只要没有commit, add后也可以用此命令查看文件修改内容

2.3 文件删除、重命名、移动

  • git rm [filename] 文件删除
  • git mv [oldname] [newname] 文件重命名
    • 重命名的本质是先复制一个份再删除一份
    • 方法一: 在编辑器中重命名之后status中有 deleted:oldname , rm olname后,状态变成 : renamed : oldname -> new name
    • 方法二:git mv old.c new.c 后,status中便有renamed : old.c -> new.c
  • git mv [filename] stuff/pretty.txt 文件移动

2.4 文件忽略

  • .gitignore

    • 将想要忽略的文件记录在此文件中
    • 已经提交的无法忽略,如果想忽略已经提交的,需要先从缓存中提出来

  • /node_modules 忽略node_modules文件夹下所有的文件

  • *.log 忽略.log结尾的文件

  • *.zip 忽略.zip结尾的文件

  • git rm -r –cached . 提出缓存中的所有文件

四、复原变更

4.1 一键还原

  • git checkout - - [file_name] 恢复到上一次的状态

4.2 撤销追踪与操作与文件还原

  • git reset HEAD [file_name] 撤销当前文件的追踪 , 将文件撤销到add状态之前
  • git checkout - - [file_name]

4.3 版本回退 v1 -> v2 -> v3 —回退到版本2—> v1 -> v2

  • git reset –hard HEAD^ 回退到上一个版本
    • –hard代表强制
    • HEAD是一个指针
  • git reset –hard HEAD^^ 回退到上上个版本
  • git reset –hard [hash号] 回退到指定的hash版本
  • git reflog 指针指向日志

4.4 回到某个版本 v1 -> v2 -> v3 —回到版本2—> v1 -> v2 -> v3 ->2

  • git log –oneline
  • git checkout [hash] – [file_name] 对一个文件回到某个版本
  • git checkout [hash] – . 将所有文件回到某个版本

五、分支合并

5.1 建立切换删除分支

  • git branch

    • 罗列所有分支名

  • git branch [name]

    • 创建一个新分支

  • git checkout [banch name]

    • 切换到某个分支
    • 在一次处理没有commit就切换到其他分支,其他分支也会发生变化, 禁止!!

  • git checkout -b [branch name]

    • 创建并切换到此分支

  • git branch -d [branch_name]

    • 删除分支

  • git branch -D [branch_name]

    • 强制删除未合并到分支

5.2 理解分支到底是什么

分支概述

  • 什么是分支

  • 分支的作用是什么

  • git branch [branch_name] [hash] 根据哈希值还原删除的分支

5.3 如何正确的合并分支

  • git merge [branch name]
    • 在master 下 git merge develop, develop的新功能会添加到master,此时master和develop内容一致
    • 在master 下 git merge develop, develop的删除的部分也会同步到master,此时master和develop内容一致

5.4 如何解决合并时发生的冲突

解决冲突

  • git merge [branch name]
  • git status 查看冲突原因
  • git merge –abort 忽略合并
  • 手动选择正确内容
  • git commit

5.5 如何通过命令查看版本路线图

  • git log
  • git log –oneline
  • git log –oneline –graph
  • git log –oneline –graph –all
    • 显示所有版本路线图
  • git log –oneline –graph -[number]
    • 最近的几个版本路线图

5.6 快转机制的意义

  • 快转机制的意义
    • 快转实际就是当前master的将来时
    • git merge branch_name –no-ff
  • master 下 git merge develop后,两者内容一致,快转机制会忽略develop版本, –no-f会保留develop版本

5.7 更多合并的方法

  • git merge [branch_name] –no-ff –no-commit
    • 需要git commit 才完成合并
  • git merge –squash [branch_name]
    • 压缩分支版本号
    • 需要git commit
  • git reset –hard ORIG_HEAD
    • 恢复到初始状态

5.8 一次性删除所有不想要的分支

  • git branch –merged | egrep -v “(^*|master|develop)” | xargs git branch -d
    • 删除除了当前分支、master分支、develop分支之外的所有已经合并的分支
  • git branch –no-merged | egrep -v “(^*|master)” | xargs git branch -D
    • 删除除了当前分支、master分支之外的所有未合并的分支
  • git branch –merged
    • 查看已经合并的版本
  • git branch –no-merged
    • 查看未合并的分支

七、多人协同开发

7.1 Push详解

  • git push -u origin 首次连接
    • 本地有一个分支master,但是远端并没有,
  • git push
    • 连接远端后,可以直接用git push推送到远端
  • git push –set-upstream origin [branch_name]
  • git push –all 提交所有分支
    • *master 本地分支 (可见)
    • remotes/origin/master 远端追踪分支 (可见)
    • github master 远端分支

7.2 Pull详解

  • git pull
    • 如果远端仓库被更新,本地再git push将会失败,因为本地没有远端的更新的部分,可以先用git pull将远端仓库内容拉取到本地,git pull后本地分支的所有内容将和远端一致,之后再本地做修改后,再git push即可
  • git remote -v 返回
    • origin git@github.com:zhuozhudd/newrepo.git (fetch)
      • fetch意为从远端拉取内容,从上往下
    • origin git@github.com:zhuozhudd/newrepo.git (push)
      • push意味从本地推到远端地址,从下往上
  • git fetch
    • 从远端拉取,但是并没有更新本地内容
    • 返回 Your branch is behind ‘origin/master’ by 1 commit, and can be fast-forwarded. (use “git pull” to update your local branch)
  • git merge
    • git fetch 后,从远端拉取但没有更新本地内容,git merge将拉取的内容和本地的合并
    • git pull = git fetch + git merge

7.3 删除远端分支、仓库迁移

  • git checkout remote_branch

    • 在本地切换到远端追踪分支

  • git push origin –delete [branch_name]

    • 在本地删除远端分支

  • git remote set-url origin [远端仓库的地址]

    • git push –all 即可迁移到远端仓库
